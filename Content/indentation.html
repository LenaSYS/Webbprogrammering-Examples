Make your code more readable

A very important aspect of writing code, any code, is to make the code readable. To do this we follow different conventions or coding standards.

If you look at https://en.wikipedia.org/wiki/Indentation_style you will find a lot of different indenting styles used.

For this course we have the following code standard

  • All code should follow K&R with Mandatory Braces Variant
  • Indention is done in increments of one (1) tab or two (2) spaces
  • Avoid unnecessary blank rows
  • Any code that is executed exactly the same number of times should be on the same indentation level, i.e., flow of execution will not disagree with the source-code indentation. This means that insertion of a new line of code anywhere is always safe as long as the braces are correctly placed.
    • Each new code block (denoted by left-hand curly brace '{') must be followed by a newline
    • The code contained in each new block (denoted by a left-hand curly brace '{') should be indented 1 level to the right of the containing block.
    • At the end of each new block (denoted by a right-hand curly brace '}') we should revert to the previous level of indentation.
    • We require mandatory braces for control statements with only a single statement in its scope, ie., bracing every if, else, while, etc., including single-line conditionals

Start of new block must be followed by newline
// Correct way to add newline
if(y>7){
  nextRow();
}

// Incorrect, no newline added after left-hand curly brace 
if(y>7){nextRow();}

Start of new block
// Correct way to indent new block
if(y>7){
  nextRow();
  y++;
}

// Incorrect way to indent new block. 2 different errors
//   1. if(y>7) and nextRow() should not be on the same indentation level since they do not exectue the same number of times
//   2. y++ and nextRow(); should be on the same indentation level since they executes same number of times
if(y>7){
nextRow();
  y++;
}

End of block
// Correct way to indent end of block
if(y>7){
  nextRow();
  y++;
}

// Incorrect way to indent new block.
//   1. does not revert to the previous level of indentation
if(y>7){
  nextRow();
  y++;
  }

Mandatory braces for control statements
// Correct way to add mandatory braces for control statements
if(y>7){
  y++;
}

// Incorrect way, no mandatory braces added for control statements
if(y>7)
  y++;

All submitted code must ad here to these code standards or you will fail on your submissions.
